# SEO Action Plan -- blog.plantdoctor.app

> Generated: 2026-03-28
> Data sources: Google Search Console (3 months), Ahrefs audit (2026-03-27), codebase analysis
> Stack: Next.js 15, Markdown/MDX, Vercel, 188 blog posts

---

## Executive Summary

The blog has strong content (188 posts, 344 unique queries, 12,170 impressions) but catastrophic conversion: 23 clicks total, 0.2% CTR, 103 pages "crawled but not indexed." Three root causes explain nearly everything:

1. **Zero external backlinks** -- Google has no trust signal to justify indexing content from an unknown domain
2. **Broken robots.txt** -- Sitemap URLs point to `news.plantdoctor.app` instead of `blog.plantdoctor.app`, so Google cannot discover sitemaps via robots.txt
3. **No internal linking** -- 95% of internal links go to the homepage; individual posts are isolated islands with no cross-linking

This plan addresses all three plus the CTR, structured data, and content quality issues identified in both reports.

### 14. Add `lastmod` from git history to sitemap

**Impact:** Low-Medium -- more accurate freshness signals
**Effort:** 2-3 hours

**Current state:** `app/sitemap.ts` uses filesystem `mtime` for `lastModified`. This resets on every deploy.

**Better approach:** Use git log to get the actual last commit date for each content file:

```typescript
import { execSync } from 'child_process'

function getGitLastModified(filePath: string): Date {
  try {
    const timestamp = execSync(
      `git log -1 --format=%cI -- "${filePath}"`,
      { encoding: 'utf-8' }
    ).trim()
    return timestamp ? new Date(timestamp) : new Date()
  } catch {
    return new Date()
  }
}
```

**File:** `/app/sitemap.ts` -- Replace `getLastModifiedFromFile()` with git-based approach.
